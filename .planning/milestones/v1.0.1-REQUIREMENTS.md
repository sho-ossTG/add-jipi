**Defined:** 2026-02-23

**Core Value:** Users can reliably request a supported episode and immediately receive a valid playable stream.

## v1.0.1 Requirements

Patch requirements for trial analytics/session controls built on top of v1.0. These requirements must preserve stream contract behavior while adding operator-visible analytics and compact daily aggregation.

### Runtime and Policy Safety

- [x] **PATCH-CTRL-01**: Stream-only gating boundaries remain unchanged; manifest and catalog routes are not blocked by session gating. (validated)
- [x] **PATCH-CTRL-02**: Runtime constants for analytics/session behavior are environment-configurable with safe defaults (`SESSION_VIEW_TTL_SEC`, `HOURLY_ANALYTICS_TTL_SEC`, existing gate constants). (validated)

### Temporary Session View

- [x] **PATCH-SESS-01**: Session view stores temporary per-session snapshots keyed by session identity with TTL expiry. (validated)
- [x] **PATCH-SESS-02**: Session view records full user-agent and trusted client IP for operational diagnostics only, with active-session index pruning by TTL. (validated)
- [x] **PATCH-SESS-03**: Session view writes occur on stream-path outcomes (hit/success/degraded/error) without changing stream response contract shape. (validated)

### Hourly Analytics Path

- [x] **PATCH-ANL-01**: Hourly analytics writes are ultra-light (`HINCRBY`/`EXPIRE` + optional `PFADD`) to minimize request-path overhead. (validated)
- [x] **PATCH-ANL-02**: Policy and stream outcomes are tracked in hourly buckets for admitted/blocked/degraded/success/error style counters. (validated)
- [x] **PATCH-ANL-03**: Hourly analytics keys expire automatically via TTL and do not become permanent storage. (validated)

### Daily Summary and Rollup

- [x] **PATCH-DAY-01**: Daily summary is stored in one permanent key with one field per day (no per-day top-level key proliferation). (validated)
- [x] **PATCH-DAY-02**: Permanent daily summary does not persist raw per-user session identifiers, IP addresses, or raw user-agent strings. (validated)
- [x] **PATCH-DAY-03**: Nightly rollup is idempotent per day and protected by a lock to avoid duplicate aggregation on concurrent trigger attempts. (validated)
- [x] **PATCH-DAY-04**: Nightly rollup aggregates hourly counters/uniques and then cleans up rolled hourly keys. (validated)

### Operator Visibility and Control

- [x] **PATCH-OPS-01**: Operator-authenticated endpoint exposes analytics visibility including current-hour counters, active session count, and available daily summary days. (validated)
- [x] **PATCH-OPS-02**: Operator-authenticated endpoint can trigger nightly rollup for a day with optional force flag for controlled re-run. (validated)

### Test and Quality Gates

- [x] **PATCH-QA-01**: New tests cover session view TTL behavior, hourly tracker behavior, and nightly rollup behavior. (validated)
- [x] **PATCH-QA-02**: Security boundary contract tests remain valid with stream-only gating behavior and include new operator analytics/rollup routes. (validated)
- [x] **PATCH-QA-03**: Required and optional gate suites pass with zero failures before patch tag creation. (validated)

## Out of Scope (v1.0.1)

| Feature | Reason |
| --- | --- |
| Public unauthenticated analytics dashboards | Security posture requires operator auth boundaries. |
| Long-term raw session history retention | Patch scope requires temporary TTL-based session view only. |
| Feature expansion beyond analytics/session controls | v1.0.1 is a targeted patch milestone, not product expansion. |
