# Plan 07-04: Daily Summary Store

## Goal

Persist compact per-day aggregates in one permanent summary key.

## Tasks

- Implement read/write/list/delete helpers for day fields.
- Keep one permanent hash key namespace for daily summaries.
- Ensure storage model is aggregate-only.

## Definition of Done

- Daily summary key model is stable and queryable.
- No raw per-user values are required in permanent records.
